## ЧАСТЬ 1 — Backend (**Ковальчук Вадим**)

### Зона ответственности
Всё в папке `backend/`

### Файлы которые нужно знать
| Файл | Зачем |
|------|-------|
| `prisma/schema.prisma` | Схема БД — пользователи, группы, предметы, оценки, лабы |
| `prisma/seed.ts` | Тестовые данные для демо |
| `src/index.ts` | Точка входа Express |
| `src/middleware/auth.ts` | JWT авторизация |
| `src/middleware/validate.ts` | Валидация через Zod |
| `src/middleware/upload.ts` | Загрузка файлов (multer) |
| `src/routes/auth.ts` | Логин |
| `src/routes/schedule.ts` | Расписание |
| `src/routes/subjects.ts` | Предметы + журнал + оценки |
| `src/routes/courseItems.ts` | Программа по предмету |
| `src/routes/labs.ts` | Сдача и проверка лаб |

### Что сделать (чеклист)
- [ ] `cd backend && npm install`
- [ ] `npx prisma migrate dev --name init`
- [ ] `npm run db:seed`
- [ ] `npm run dev` — проверить http://localhost:3001/api/health
- [ ] Протестировать POST `/api/auth/login` через Postman/Thunder Client
- [ ] Убедиться что роли STUDENT/TEACHER ограничивают доступ (403 без прав)

### Как запустить локально
```bash
cd backend
npm install
copy .env.example .env
# Создай БД Gradebook в SSMS (database/init.sql) — см. docs/SSMS_SETUP.md
npx prisma migrate dev --name init
npm run db:seed
npm run dev
```

### Что рассказать на защите
- Почему SQL Server + SSMS + Prisma (учебный стандарт, удобный GUI, полноценная SQL СУБД)
- Как работает JWT (токен в header `Authorization: Bearer ...`)
- Схема БД: User, Group, Subject, LessonDay, Grade, CourseItem, LabSubmission
- Валидация на сервере (Zod)
- Хранение файлов в `uploads/`

### Возможные доработки (если есть время)
- Добавить авто-опоздание по `LessonTiming` (тайминги уже в БД)
- Показать таблицы в SSMS на защите (Select Top 1000 Rows)
- Добавить refresh token

## ЧАСТЬ 3 — Frontend: Преподаватель — Журнал (**Богданов Глеб**)

### Зона ответственности
Интерактивный журнал преподавателя

### Главный файл
`frontend/src/pages/JournalPage.tsx` — **самый важный файл лабы**

### Функционал по ТЗ (уже реализован)
| Действие | Как |
|----------|-----|
| Добавить день | Кнопка «Добавить день» |
| Оценка | ЛКМ → диалог или клавиши 2-5 |
| Опоздание | СКМ (средняя кнопка мыши) |
| Отсутствие | ПКМ |
| Отчисленный студент | Строка обесцвечена, opacity 0.6 |
| Новый студент | Голубая подсветка + chip «новый» |
| Подсветка при наведении | Строка + столбец + пересечение |

### Что сделать (чеклист)
- [ ] Войти как `teacher@university.ru` / `password123`
- [ ] Журнал → Веб-разработка
- [ ] Добавить день — появился новый столбец
- [ ] ЛКМ на ячейку → выставить оценку 4
- [ ] ПКМ → «Н» (отсутствие)
- [ ] СКМ → «О» (опоздание) — *на тачпаде может не работать, используй мышь*
- [ ] Навести мышь — подсветка строки/столбца
- [ ] Проверить student4 (отчислен) и student1 (новый)

### API которые использует журнал
```
GET  /api/subjects/:id/gradebook?groupId=...
POST /api/subjects/:id/days        { groupId, date, topic }
POST /api/subjects/:id/grades      { lessonDayId, studentId, type, value }
```

### Что рассказать на защите
- Логика `handleCellClick` — разные кнопки мыши
- Валидация оценок (только 2-5)
- CSS подсветка через state `hover`
- Разграничение: студент видит журнал read-only, преподаватель — edit

### Возможные доработки

## Если что-то сломалось

| Проблема | Решение |
|----------|---------|
| CORS ошибка | Backend должен быть на :3001, frontend на :5173 |
| 401 Unauthorized | Перелогиниться, проверить token в localStorage |
| Пустой журнал | `npm run db:seed` в backend |
| Файл не грузится | Проверить папку `backend/uploads/` существует |
| Prisma ошибка | `npx prisma migrate dev` |
